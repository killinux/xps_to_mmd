## What's not perfect (known gaps — v1.1 candidates)

### 1. Wrist drift 6.1% vs target (Inase)

- Frame 0 baseline drift = 2.4% (armature height diff: 1.689m vs 1.682m)
- VMD path adds ~3.7% peak at frame 60
- Phase D shows 頭 direction delta 67°, 手首 27°, 腰 25° — these rest-pose Y-axis direction differences come from hardcoded bone.tail directions in `complete_bones_operator.py`:
  - 頭.tail = head + (0, 0, bone_length*0.25) — straight up; target PMX likely points forward (+Y)
  - 手首.tail — short stick, direction varies
- **Fix (v1.1)**: adjust 頭/手首/腰 tail directions in `complete_bones_operator.py` to match MMD standard forward-pointing convention. Low-risk localized edit.

### 2. Reika breast only 1 rigid (expected pair L+R)

- Reika `boob right/left N` should match keywords but only 1 detected
- `_find_breast_bones` requires parent ∈ {上半身, 上半身2, chest, Chest}. Reika breast bone parent might be different (e.g. `Chest` capitalized or something post-convert).
- **Fix (v1.1)**: print bone names + parents during detection for diagnostic; broaden parent candidates if needed.

### 3. Phase C hair_tip_delta 26cm on Reika

- 14.7% of armature height. Larger than Inase 1.1cm.
- Reika has 13 hair chains (vs Inase 6) and very long hair.
- No explosion (stable numbers), so not physics broken. Just natural tip hanging.
- **No fix needed** unless visual bake screenshot shows issue.

### 4. Target PMX vs ours has 2x bones (263 vs 145)

- Target Purifier Inase has accessory/clothing/IK helper bones we don't generate
- Non-blocking for VMD playback since VMD only keys core bones
- **No fix needed** for v1
